# Microsoft To-Do 웹 서비스 구축 기록

본 문서는 `~/apps/mstodo_web` 디렉토리에 구축된 Microsoft To-Do 웹 서비스의 구성 및 운영 방법을 상세히 기록합니다.

## 1. 개요
기존에 존재하던 Microsoft To-Do 연동 스크립트(`app.py`)를 기반으로, 사용자가 웹 브라우저를 통해 할 일을 확인하고 관리할 수 있도록 Docker 컨테이너 기반의 웹 서비스를 구축하였습니다.

## 2. 주요 변경 사항
- **디렉토리 이름 변경**: `~/apps/mstodo` → `~/apps/mstodo_web` (작업 명확성 확보)
- **컨테이너화 (Docker)**: 환경 격리 및 배포 용이성을 위해 Docker 적용
- **서비스 안정화**: Flask 내장 서버 대신 `gunicorn`을 사용하여 프로덕션급 웹 서버 환경 구축
- **자동 실행 설정**: 시스템 재부팅 시에도 서비스가 자동으로 시작되도록 설정

## 3. 기술 스택 및 환경
- **언어**: Python 3.11 (slim 이미지 사용으로 최신 라이브러리 호환성 및 경량화 확보)
- **웹 프레임워크**: Flask
- **서버**: Gunicorn (포트 5001)
- **라이브러리**: `pymstodo` (Microsoft To-Do API 연동)
- **오케스트레이션**: Docker Compose

## 4. 파일 구성 정보
- **Dockerfile**: Python 3.11 환경에서 필요한 라이브러리를 설치하고 앱을 실행하는 레시피입니다.
- **docker-compose.yml**: 컨테이너의 이름, 포트(5001), 볼륨 마운트, 환경 변수(서울 시간대), 재시작 정책(`always`)을 정의합니다.
- **requirements.txt**: `Flask`, `pymstodo`, `gunicorn` 등 서비스 실행에 필요한 패키지 목록입니다.
- **config.ini / token.json**: Microsoft To-Do API 인증 정보가 저장된 기존 파일들입니다.

## 5. Nginx Proxy Manager(NPM) 연동 정보
외부에서 접속하기 위해 NPM에서 다음과 같이 Proxy Host를 설정하십시오.

- **Scheme**: `http`
- **Forward HostName / IP**: `10.0.0.99` (호스트의 내부 IP)
- **Forward Port**: `5001`
- **Websockets Support**: 필요 시 활성화 (기본 기능에는 필수 아님)

## 6. 운영 및 관리 명령어
모든 명령어는 `~/apps/mstodo_web` 디렉토리 내에서 실행해야 합니다.

- **서비스 상태 확인**: `docker-compose ps`
- **실시간 로그 모니터링**: `docker logs -f mstodo_web`
- **코드 수정 후 반영 (재빌드)**: `docker-compose up -d --build`
- **서비스 완전히 중지 및 제거**: `docker-compose down`

## 7. 기타 참고 사항
- `app.py`에서 정의된 포트는 `5001`이며, 컨테이너 내부와 호스트 모두 동일하게 `5001` 포트를 사용하도록 매핑되었습니다.
- Microsoft API 토큰 만료 시 `token.json` 갱신이 필요할 수 있습니다.
